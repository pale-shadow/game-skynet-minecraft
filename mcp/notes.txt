### **Phase 1: Environment Setup on Stargate**
Since Stargate is running **Debian 12** and features a large NVMe mount at `/mnt/clusterfs2`, you should move the primary workspace there to ensure maximum I/O performance for your **T2BM (Text to Building in Minecraft)** model weights [92, Conversation].

1.  **Clone and Prep:**
    ```bash
    cd /mnt/clusterfs2
    git clone https://github.com/pale-shadow/game-chonk-minecraft.git
    cd game-chonk-minecraft
    # Create a fresh venv on the NVMe
    python3 -m venv venv
    source venv/bin/activate
    pip install -r requirements.txt
    ```
2.  **Verify Hardware Exports:** Ensure the `direnv` exports for **CUDA** and **LD_LIBRARY_PATH** are active to support NPU-accelerated inference [Conversation].

### **Phase 2: Setting up the MCP Host & Servers**
Deploying MCP on Stargate allows your **Unified Brain** to act as a "Host" that can call specialized "Tools" to interact with the Minecraft world [32, Conversation].

1.  **Install the MCP Python SDK:**
    `pip install mcp`
2.  **Configure Essential MCP Servers:**
    *   **Filesystem Server:** Grants the T2BM model access to your `schematics/` and `models/` directories to read "Noodle Logic" blueprints and write new schematics [34, Conversation].
    *   **Git Server:** Automates the **`backup_to_git.sh`** workflow, allowing the AI to commit new "Void-Tech" designs after successful deployment.
    *   **RCON Tool Server:** Wraps your `SkynetRCON` client into an MCP tool. This allows the LLM to execute `setblock` or `fill` commands directly via standardized protocol calls [34, Conversation].
    *   **Database Tool:** Connects to your **`database/`** folder to let the AI "remember" the location of **Permanent Cultural Heritage Zones** like the **2012 Washington Station** [34, Conversation].

### **Phase 3: Migrating and Updating the Orchestrator**
