# Hub 07: Neural-Data Vault (MariaDB)
This server, designated as Hub 07 (Data Haven), provides the centralized "Neural-Data Vault" required for the high-performance orchestration of the Bitsmasher "Void-Tech" infrastructure
.
System Overview
Hostname: blowfish
Operating System: OpenBSD 7.8
Internal IP: 10.10.12.15
Role: Primary SQL Storage for Skynet Intelligence and Map Telemetry.
Functional Roles
The MariaDB instance on blowfish is responsible for the following critical data streams:
AI Intelligence Logs: Serves as the "deep-storage cold cell" for all encrypted AI logs generated by the Skynet Inference Nexus (Hub 01) during reactive mutations
.
Sense/Net Archives: Stores surveillance data and environmental analysis captured by the Skynet Mono-Eye Sensor (Hub 06) Edge TPU vision node
.
BlueMap SQL Storage: Configured as the persistent storage backend (storage: "sql") for the Overworld and Nether mapping renders, replacing standard file-based storage for improved query performance
.
Plugin Data: Houses backend data for "most essential" plugins like CoreProtect, ensuring that every block change in the Skynet AI Testing Field is logged for immediate "Void-Tech" recovery
.
Security Configuration
Following the established Minecraft Server Security Best Practices, this instance is hardened with the following measures:
Access Control: Database access is strictly restricted to the Stargate MCP server and relevant hardware nodes within the 10.10.x.x subnet
.
Password Policy: Utilizes strong, unique credentials for the system panel and database users to prevent unauthorized entry
.
OpenBSD Hardening: Leverages the security-first architecture of OpenBSD 7.8 to mitigate backend infrastructure exposure
.
Backup & Technical Ledger
While this MariaDB instance stores live data, its state is periodically synchronized with the Technical Ledger (GitHub) via the following process:
Backup Script: The bin/backup_to_git.sh script on the Chonk host triggers data exports that are committed to the game-server-minecraft repository
.
Automated Daily Backups: In accordance with server stability protocols, the database is backed up daily with off-server storage to protect against file corruption or accidental deletions
.
Connectivity
Daemons across the Bitsmasher network (specifically those residing in the @servers/ directory) are configured to communicate with the Stargate MCP server, which then facilitates necessary queries to this vault at 10.10.12.15.
